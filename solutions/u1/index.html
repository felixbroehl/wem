1.1. Fachliche Argumentation über Erfolgsprinzipien des WWW

Mit welchen fachlichen Argumenten wurde das WWW-Proposal von TBL abgelehnt?

Chef von Tim-Berners-Lee lehnte das Konzept mit folgendem Kommentar ab: "Interessant, aber ein bisschen vage" Ablehnung seines Konzeptpapiers 1991 von weltgrößter Hypertext-Konferenz, da sein Konzept angeblich nicht so weit sei wie andere und Broken-Links beinhalten würde.

Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?

Grund für den Erfolg war, dass er sein Konzept am Rande der Veranstaltung mit einem Poster vorstellen durfte und dabei von Marc Andreesen entdeckt wurde und mit diesem zusammen der Browser Mosaic entwickelt wurde. 1994 gründete TBL das WWW-Consortium. Fachliche Gründe waren, die Offenheit, Dezentralität und Evolvierbarkeit des WWW.

Was wäre der Preis für die garantierte Verhinderung von “broken links”?

Referenzielle Integrität => Zentrale Kontrolle, Geschlossenheit

1.2. HTTP

Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?

OK, Anfrage wurde erfolgreich durchgeführt

Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?

Moved Permanently, Seite wurde permanent verschoben. Weiterleitung über Location-Header.

Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?

Bad Request, Anfrage des Nutzers hatte eine für den Server nicht verständliche oder unvollständige Syntax. Somit ist die Anfrage entsprechend der vom Server erwarteten Syntax anzupassen.

Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?

Forbidden, Der Nutzer ist nicht berechtigt auf die aufgerufene URL zuzugreifen. Somit müsste sich ein Nutzer zunächst authentifizieren.

In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performanzgründen soll die Abfrage jedoch cacheable sein. Wie könnten Sie dafür eine Lösung angehen?

Mit Cache-Control-Headern, jedoch dürfte der Browser diese für OPTIONS-Anfragen ignorieren.
